# SEO And Performance Plan

## Local SEO

- Restaurant schema is embedded in `app/layout.tsx`.
- Per-page metadata is defined for Home, Menu, Order, About, Reviews, Contact, and Admin.
- `/sitemap.xml` is generated by `app/sitemap.ts`.
- `/robots.txt` allows customer pages and blocks `/admin` and `/api`.
- Google Business Profile URL is configured with `NEXT_PUBLIC_GOOGLE_BUSINESS_URL`.
- Google Maps embed is configured with `NEXT_PUBLIC_GOOGLE_MAPS_EMBED_URL`.

## High-Intent Keywords

- Homemade Pakistani food delivery
- Homemade handi delivery
- Zinger burger delivery
- Pakistani gravy delivery
- Foodpanda alternative direct order
- Local homemade food delivery

## Performance Targets

- Mobile Lighthouse above 90.
- Main pages render server-side.
- Images are served from `public/images/whatsapp` and optimized by Next.js.
- Interactive JavaScript is limited to cart, menu filters, checkout, review form, and admin controls.
- Sticky CTAs use simple CSS and icons with no animation dependency.

## Launch Checks

- Compress any new food images before adding them.
- Keep hero image under 300 KB where practical.
- Run `npm run build` before deployment.
- Run Lighthouse mobile after deployment and replace oversized assets if LCP is above 3 seconds.
